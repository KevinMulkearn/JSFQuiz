Scopes of Managed Beans:

@RequestScoped
A request scoped bean lives the duration of a single HTTP request/response cycle.
It is used for simple and non-ajax forms and presentations.

@SessionScoped
A session scoped bean lives the duration of the established HTTP session
and exists across multiple HTTP requests.
It is used for client specific data, such as the logged-in user and user preferences.

@ApplicationScoped
An application scoped bean lives the duration of the web application's run time,
it exists for all users interacting with the web application.
It is used for application wide data/constants, such as a dropdown list which is the same for all users.

@ViewScoped
A view scoped bean lives as long as the user is interacting with the same JSF view
by postbacks which call action methods returning null without any navigations or redirects.
It is used for rich ajax-enabled dynamic views.

@NoneScoped
A none scoped bean lives the duration of a single EL (Expression Language) evaluation
(the bean is destroyed immediately after evaluation).

@CustomScoped
A custom scoped bean can be defined in custom map by the user.
It is used for special cases only.

Three Forms of JSF Page Navigation:

The there main types of navigation JSF page are:

  • Implicit Navigation
  • Explicit Navigation
  • Auto-Navigation

Implicit Navigation:

  • Value ‘aaa’ of action attribute implicitly resolved to ‘aaa.xhtml’
  • There is no need to declare navigation rules
  • The outcome can be placed in the action attribute
  • There is less overhead when looking up the outcome for a action.

Explicit Navigation:

  • Navigation requires the use of explicit navigation cases
  • Explicit navigation rules can be defined in faces-config.xml
  • Only one location is used to set all navigation rules
  • Maps value from sources to specific targets
  • Need one mapping per page
  • Same value from different source can map to different target

Auto-Navigation:

  • String value ‘aaa’ of any bean interaction automatically resolved to ‘aaa.xhtml’
  • Useful for conditional navigation
  • very flexible conditional navigation rule to solve the complex page navigation flow
